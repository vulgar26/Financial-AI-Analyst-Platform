# 产品愿景与路线图（北极星文档）

> 本文是项目的定位基准。当其它文档与本文冲突时，以本文为准。
> 最后更新：2026-06（产品方向重定为「长期价值投资研究工作台」）

---

## 一句话定位

一个帮你**想清楚**长期价值投资决策的研究工作台：你把投资原则写成可验证的规则，
系统持续帮你检查、给依据、提示风险、对照历史——**最终判断由你自己下**。
它把你的经验沉淀成规则，规则长期验证、迭代。

它**不是**：自动选股、投资建议、自动交易、收益承诺、又一个金融问答机器人。

---

## 四个定调（不可动摇的设计基准）

1. **角色：帮你想清楚，不替你决策。**
   系统给「判断 + 依据 + 风险 + 历史参照」，但买卖与否你来定。
   它是思考放大器、魔鬼代言人，目的是训练**你自己**的判断力。
   → 这砍掉了合规风险、黑箱信任、「错了谁负责」三大难题。

2. **尺度：先只做长期价值投资（月~年）。**
   中短期波段、事件驱动放进长期计划。
   → 长期价值投资的数据（财报、估值、基本面）便宜、变化慢、易回测，
   不需要实时行情，是个人项目唯一可行的起点。

3. **灵魂：把原则变规则，规则变工作流，工作流持续验证。**
   「A 发生 → 看 B 指标 → B 满足 → 关注 C → 长期观察对错」。
   区别于「问一句答一句」的关键：**沉淀**与**持续验证**。

4. **形态：投资研究工作台，对话只是入口之一。**
   核心对象是「标的」和「规则」，不是聊天框。详见下方「形态」。

---

## 形态：为什么不是纯对话 AI

纯对话是一次性、无状态、留不下东西的——而本系统的价值恰恰在**沉淀**与**持续**。
最终形态是一个工作台，由这几部分组成：

| 部分 | 作用 | 现状 |
| --- | --- | --- |
| **对话 / 问答** | 临时探索的入口（「分析下苹果基本面」） | 已有（SSE 链路） |
| **标的档案 / Watchlist** | 每个关注标的一个持续更新的档案：基本面、你的判断、触发的规则、历史表现 | 设计参考见 archive/finance-observer-boundaries |
| **规则 / 原则库** | 你写下的投资原则，系统对关注标的持续检查 | 未做（愿景灵魂） |
| **定时巡检 / 主动提醒** | 自动跑规则、发现触发/新财报/重大变化时主动推送 | 已有异步任务框架骨架（AgentTaskWorker，当前 mock） |
| **评测 / 验证** | 验证规则准不准、改了会不会变差（复用 eval 系统） | 已有 eval 系统 |

**形态是结果，不是起点。** 先有真实的研究行为发生在系统里，工作台的样子由里面装的东西决定。
所以**不预先搭工作台框架**，先用最小形态跑通价值链。

### 规则库的归属边界（重要认知底座）

规则**以你自己写为主，AI 辅助提炼，永远由你拍板**：

- ❌ 不让 AI 全自动总结规则：会过拟合（历史完美、未来失效）、失去学习意义、无法负责。
- ✅ AI 的角色：帮你把模糊想法形式化、当魔鬼代言人挑盲区、基于数据给「候选观察」（不是「规则」）。
- 规则库是**你的**大脑，AI 是打磨想法、挑刺、找资料的助手。

## 现有工程基础如何对口这个愿景

过去 2.5 个月的工程积累并非白做，它们恰好是这个愿景的零件：

| 已有能力 | 对应愿景中的角色 |
| --- | --- |
| `PLAN → RETRIEVE → TOOL → GUARD → WRITE` 固定链路 | 决策流水线 + 风险门控 |
| eval 评测系统（确定性回归） | 规则验证引擎（「历史上准不准」的雏形） |
| 异步任务框架（AgentTaskWorker） | 定时巡检 / 主动提醒的基础 |
| 工具治理（超时/限流/熔断/降级） | 真实数据源接入的安全保障 |
| 合规边界（不编造、证据不足澄清、数据时点标注） | 「依据是什么、风险是什么」的工程保障 |
| 知识库 / 用户画像 | 标的档案 / 偏好记忆 |

## 金融能力的覆盖层次（学习脚手架）

从窄到宽，**系统的扩展速度应匹配使用者金融认知的成长速度**：

1. **公司层面**（最先，最具体）：财务健康、估值、护城河 ← **第一块砖在这层**
2. **行业层面**：增长/衰退、竞争格局
3. **宏观层面**（最难、最易空谈）：利率、通胀、周期 ← 放到很后面
4. **认知/行为层面**（最被低估）：对抗自己的偏见、情绪、冲动 ← 「帮你想清楚」主要在这层发力

## 落地路线图（砖块式，每块独立可用、可验证）

> 原则：先用最小闭环跑通一条价值链，再被真实需求驱动着长出框架。
> 严禁「先搭漂亮框架再填内容」——那是本项目最大的历史陷阱。

### 第一块砖：接入真实基本面数据（进行中）

把 mock `MarketDataTool` 换成真实美股基本面数据源（候选 FMP / Alpha Vantage）。
- 输入股票代码 → 返回真实股价、PE/PB、市值、营收、利润、负债等
- 复用现有工具治理（超时/熔断/降级）
- 标注数据来源与时点（合规边界终于有真数据可标）
- 验证：问「分析 AAPL 基本面」，返回真实数据 + LLM 解读 + 风险提示
- 市场选择：先美股（免费数据质量高、LLM 最熟、价值投资教材多）；A 股等闭环跑通后再加

### 第二块砖：LLM 基于真实数据做结构化解读

估值高低、盈利质量、负债风险，带数据时点和来源。
对应旧设计 `financial-workflows-v1.md` 的 `company_snapshot` / `market_data_explain` 场景。

### 第三块砖：历史财务数据 + 「历史分位」

接入历史财务数据，回答「这只票 PE 在它过去 5 年处于什么分位」。
这是「历史参照 / 历史上准不准」的第一步。

### 第四块砖起：第一条规则 → 规则验证 → 标的档案 → 巡检

- 写第一条成文规则（判断 + 依据 + 风险 + 历史）
- 用 eval 系统验证规则
- 标的档案沉淀（Watchlist，参考 finance-observer 设计）
- 规则多了 → 自然需要规则管理/巡检 → 那时才搭对应框架

### 长期计划（明确推迟，不现在做）

- 中短期 / 事件驱动尺度
- A 股、基金、宏观数据、新闻情绪
- PDF 研报 / 财报解析
- 多 agent、事件溯源等重型架构（详见 archive/ 中的历史设计，**不作为当前指引**）

## 合规与安全边界（继承自现有设计，不可破坏）

- 内容为研究 / 教育用途，**不构成投资建议**
- 不给个性化买入 / 卖出 / 持有建议，不承诺收益，不预测确定性价格
- 不执行交易、开户、KYC，不使用非公开重大信息
- 行情 / 财报 / 新闻必须尽量标注来源和时点
- 检索为空 / 工具失败 / 证据不足时降级说明，不编造
- mock / 非实时数据必须显式标注

金融 guard 的结构化归因码（来自 `financial-workflows-v1.md`，落地时复用）：
`FINANCE_INVESTMENT_ADVICE_BLOCKED`、`FINANCE_MOCK_DATA_DISCLOSURE_REQUIRED`、
`FINANCE_EVIDENCE_INSUFFICIENT`、`FINANCE_ENTITY_AMBIGUOUS`、
`FINANCE_TOOL_UNAVAILABLE`、`FINANCE_ACCOUNTING_LEGAL_BOUNDARY`、`FINANCE_FRESHNESS_UNKNOWN`
